> Task of the Day: Study and Implement Safety measures and fallbacks, and Guardrails for agents.

---
I touched on the following today:
* Input Guardrails: These are the measures set up in place to make sure that the Agent or LLM is aware of the security format of it's output response.
* Output Guardrails: These are measures applied to the Agent's response to make sure that the input guradrails were respected and to enforce other measures.
* Action confirmation: The Agent needs to confirm its next action with a user or Super Agent which will review its work or output because giving it the go ahead.
* Rate limiting: Sets a limit on the number of calls an agent can make over a certain period
* Scope Limiting: Limits the Agent from accessing particualr tools.
